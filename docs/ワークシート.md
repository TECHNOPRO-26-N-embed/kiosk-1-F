## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* MAX在庫数：50（商品の入れ替えあり）
* セキュリティ面:できる限り考慮、ただし止まらないように
* 商品の種類:50
* 残すデータ：故障予知のデータ？
* 金額の表示：いる
* 画面はテキストメニュー　かっこよく
* 商品の選択、数量入力、合計選択ができるように
* 操作ログをCSVファイルに残したい
* 複数を客が使える

### 2.2 機能要件

| 機能ID | 機能名             | 概要                                                    |
|:------:|:------------------|:--------------------------------------------------------|
| F01    | 商品一覧表示         | 登録されている商品の一覧（商品名・価格・在庫数）を表示する |
| F02    | 商品登録             | 新しい商品を登録する                                   |
| F03    | 商品情報編集         | 既存商品の情報（商品名・価格・在庫数）を編集する          |
| F04    | 商品削除             | 商品を削除する                                         |
| F05    | 在庫数自動更新       | 購入時に在庫数を自動で減算する                           |
| F06    | 商品入れ替え         | 商品の入れ替え（新商品への差し替え）に対応する            |
| F07    | 商品選択             | ユーザーが購入する商品を選択する                         |
| F08    | 数量入力             | ユーザーが購入する数量を入力する                         |
| F09    | 合計金額表示         | 選択商品・数量に応じた合計金額を表示する                  |
| F10    | 現金投入受付         | ユーザーから現金を受け付ける                             |
| F11    | 釣銭計算・返却       | 必要に応じて釣銭を計算し返却する                          |
| F12    | 購入確定             | 購入内容を確定し、在庫・売上・ログを更新する              |
| F13    | 複数客対応           | 複数のユーザーが同時に利用できるようにする                |
| F14    | ログ保存             | 販売履歴や操作ログ、故障予知用データをCSV形式で保存        |

### 2.3 非機能要件

| 区分 | 要件 |
| :--- | :--- |
| 可用性 | 入力異常やログ書き込み失敗が発生しても、プログラム全体は終了せずメニューに復帰できること。 |
| 性能 | 通常操作（メニュー表示、商品選択、購入確定）の応答は1秒以内を目標とする。 |
| 保守性 | 商品数上限（50件）や税計算方式などの定数を、1か所の定義変更で反映できる構造にする。 |
| ログ品質 | 全ログに日時、操作種別、結果（OK/NG）、セッションIDを含めること。 |
| 操作性 | 初見利用者でも、3ステップ以内（選択→数量→投入）で購入完了できること。 |

### 2.4 制約条件・前提

| 項目 | 内容 |
| :--- | :--- |
| 商品種類数 | 同時に管理できる商品は最大50種類。 |
| 在庫数 | 各商品の在庫は0以上50以下。 |
| 入力方式 | キーボードによる数値入力を基本とし、無効入力時は再入力させる。 |
| データ保存 | 永続化はCSVファイルで行い、文字コードはUTF-8を使用する。 |
| 同時利用 | 複数利用者はセッション単位で順次処理し、同一在庫への更新競合を防ぐ。 |

### 2.5 受け入れ基準（完了判定）

| ID | 受け入れ基準 |
| :--: | :--- |
| A01 | 商品登録時、51件目の追加は拒否され、エラーメッセージが表示される。 |
| A02 | 在庫を超える数量入力は確定されず、再入力を求める。 |
| A03 | 購入確定時に合計金額と釣銭が正しく表示され、在庫が減算される。 |
| A04 | 1取引ごとに販売ログCSVと操作ログCSVへ1行以上追記される。 |
| A05 | 数字以外入力や不正商品ID入力でもプログラムが停止せず、メニューに戻れる。 |
| A06 | エラーログには故障予知用としてエラー種別と発生時刻が記録される。 |

## 3. 基本設計

### 3.1 画面遷移図

```
メインメニュー
   → [1] 商品購入
      → F01 商品一覧表示
      → F07 商品選択
      → F08 数量入力
      → F09 合計金額表示
      → F10 現金投入受付
      → [金額不足] 追加投入画面へ
      → F12 購入確定（F05在庫数自動更新・F11釣銭計算・F14ログ保存）
      → 完了画面（購入金額・釣銭表示）
      → メインメニュー

   → [2] 商品管理（管理者）
      → F02 商品登録
      → F03 商品情報編集
      → F04 商品削除
      → F06 商品入れ替え
      → 在庫上限チェック（MAX50）
      → F14 ログ保存
      → メインメニュー

   → [3] ログ確認
      → F14 ログ保存・表示
      → メインメニュー

   → [9] 終了
```

### 3.2 データ設計

| データ名 | 機能要件対応 | 内容・説明 |
| :--- | :--- | :--- |
| 商品マスタ | F01,F02,F03,F04,F06 | 商品ID、商品名、価格、在庫数、有効/無効（最大50件） |
| 取引情報 | F05,F07,F08,F09,F10,F11,F12 | 取引ID、セッションID、商品ID、数量、合計金額、投入金額、釣銭、日時 |
| 操作ログ | F14 | ログID、セッションID、日時、操作種別、詳細、結果、エラーコード |
| 故障予知データ | F14 | 記録ID、日時、エラー種別、発生箇所、入力値、メッセージ、重大度 |
| セッション情報 | F13 | 複数利用者を区別するための利用単位情報 |

### 3.3 CSV保存形式

- 販売ログCSV: 取引ID,セッションID,日時,商品ID,商品名,単価,数量,小計,投入金額,釣銭,結果
- 操作ログCSV: ログID,セッションID,日時,操作種別,詳細,結果,エラーコード
- 故障予知CSV: 記録ID,日時,エラー種別,発生箇所,入力値,メッセージ,重大度

### 4. 詳細設計（アルゴリズム）

#### 4.1 メインループ

```
1. メインメニューを表示（F01, F13）
2. ユーザー入力受付
3. 入力に応じて分岐
   - 1: 購入処理（F07〜F12）
   - 2: 商品管理（F02〜F06）
   - 3: ログ表示（F14）
   - 9: 終了
   - その他: エラー表示・ログ記録（F14）
4. メインメニューに戻る
```

#### 4.2 個別機能アルゴリズム

- F01: 商品一覧表示 → 商品マスタから有効商品を一覧表示
- F02: 商品登録 → 入力値検証・上限チェック後、商品マスタに追加
- F03: 商品情報編集 → 対象商品を検索し、項目を更新
- F04: 商品削除 → 論理削除（有効→無効）
- F05: 在庫数自動更新 → 購入確定時に在庫数減算
- F06: 商品入れ替え → 既存商品を新商品で上書き
- F07: 商品選択 → 商品ID入力・存在チェック
- F08: 数量入力 → 1以上かつ在庫以下か検証
- F09: 合計金額表示 → 単価×数量を計算
- F10: 現金投入受付 → 合計金額以上になるまで受付
- F11: 釣銭計算・返却 → 投入金額-合計金額を計算
- F12: 購入確定 → 在庫・売上・ログを更新
- F13: 複数客対応 → セッションIDで識別し直列化
- F14: ログ保存 → 各操作・エラーをCSVに記録

#### 4.3 エラーハンドリング

| 異常系シナリオ | 機能要件 | 対処方針 |
| :--- | :--- | :--- |
| 数字以外の入力 | F07,F08,F10 | エラー表示・再入力、ログ記録 |
| 金額不足 | F10 | 不足金額表示・追加入力・キャンセル可 |
| 存在しない商品ID | F07 | エラー表示・商品一覧に戻す・ログ記録 |
| 在庫超過入力 | F08 | 上限表示・再入力・連続失敗時はメニュー復帰 |
| CSV書き込み失敗 | F14 | 画面継続・メモリ退避・次回再試行 |
| 商品数上限超過 | F02 | 登録拒否・上限超過表示・入れ替え案内 |
| 削除/入れ替えID不一致 | F04,F06 | エラー表示・再選択 |
| セッション競合 | F13 | 直列化・再試行・メニュー復帰 |

### 5. 備考・メモ

* 各機能IDごとに設計・テスト観点を明確化し、要件変更時は全体を一貫して見直す
* 商品削除・入れ替えは論理削除（有効→無効）を基本とし、履歴管理を徹底する
* ログ保存・エラー記録は必ず時刻・セッションID・操作種別を含める
* 入力値検証は「空文字→数値→範囲」の順で統一
* CSVはヘッダー必須・なければ自動作成
* 取引ID/ログIDは日時+連番で一意化
* 複数利用者はセッションIDで区別、在庫更新は直列化
* 異常時も停止せず、致命的でなければメニュー復帰
