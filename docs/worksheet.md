### 2.2 機能要件

| 機能ID | 機能名 | 概要 |

## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. メインメニューを表示する
2. ユーザーの入力を受け取る

   - 1の場合: （処理を記述）
   - 2の場合: （処理を記述）
   - 9の場合: プログラムを終了する
4. メインメニューに戻る（1へ）
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: （機能名）

```
| F01 |商品の一覧表示 |商品の表示/不足硬貨の表示(ランプが光る) |

1.商品一覧の表示　"""商品の表示"""

2.if(売り切れていたら)　"""売り切れと表示"""

3.金投入待ち　"""金が入れられたらF02に移行"""

4.金額が足りないとき """F02から情報を受け取り、足りない金額分を表示"""

5.返金処理が来た場合　"""1に移行"""

6.購入処理が終了したら　"""1に移行"""
```

#### 機能2: （機能名）

```
F02
入金の検知→入金額の計算（加算）→計算結果を表示
```

#### 機能3: （機能名）

```
商品のボタンが押される→販売可能商品（金額と在庫）かの判定→OKならF04に移行
                                                         →NGならメッセージ表示
```

#### 機能4: （機能名）

```
F04: 商品購入処理

選んだ商品が売り切れでないか確認
個数と合計を計算
入金額が足りてれば購入成立
在庫を減らしてログに残す
余ったらおつり処理へ
```

#### 機能5: （機能名）

```
| F05 |ログ |

1.購入させた商品を記録して
商品を-1する。
```

#### 機能6: （機能名）

```
| F06 |返金処理 |余ったお金を戻す(おつり) /
 購入をしない時戻すそして1に戻る|

1.余ったお金を戻す(おつり)
投入された金額から購入された商品の金額の差を求める
差の値をおつりとして出力する

2.購入をしない時戻すそして1に戻る
前提:キャンセルを入力
投入された金額をおつりとして出力する
```

#### 機能7: （機能名）

```
1.商品の入れ替え
入れ替え元の商品の情報をCSVから取得する
入れ替え元の商品の情報と新しい商品の情報に更新する

2.商品の補充
商品の情報が登録されているCSVの全ての商品の商品在庫数を50に更新する
```

#### 機能8: （機能名）

```
F08: おつり不足対応

おつりを計算
硬貨の在庫で渡せるか確認
足りなければ「おつり用意できません」と表示して返金or中断
渡せるならおつりを出すだけ
```
