## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）


> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* MAX在庫数：50（商品の入れ替えあり）
* セキュリティ面:できる限り考慮、ただし止まらないように
* 商品の種類:わからん
* 残すデータ：故障予知のデータ？
* 金額の表示：いる 
* 画面はテキストメニュー　かっこよく
* 商品の選択、数量入力、合計選択ができるように
* 操作ログをCSVファイルに残したい
* 複数を客が使える
* 

### 2.2 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 |商品の一覧表示 |商品の表示/不足硬貨の表示(ランプが光る) |
| F02 |入金処理 |入れられた硬貨の換算と計算と処理,金額の表示 |
| F03 |商品の選択 |商品を選択 /売り切れた商品は選択を無効にする|
| F04 |商品の購入処理 |入金したお金を使い商品を購入する |
| F05 |ログ |購入した商品をCSVに記録する |
| F06 |返金処理 |余ったお金を戻す(おつり) / 購入をしない時戻すそして1に戻る|
| F07 |商品入れ替え |商品の入れ替え・補充 |
| F08 |おつり不足時対応 | メッセージ(またはアラーム) |
